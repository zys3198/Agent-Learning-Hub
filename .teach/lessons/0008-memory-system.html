L0008 - 记忆系统设计

短期记忆、会话记忆、长期记忆——三层架构让 Agent 从"金鱼脑"变成"老同事"

L2.3 · 记忆系统设计

1. 为什么 Agent 需要记忆？

L0004 的 Agent 每次对话从零开始，没有记忆。这意味着：

  • 你说过"我是 Java 开发者"，下一次提问它忘了
  • 你让它查了 A，它查了，但 5 轮后再问 A 它又查一遍
  • 关掉终端重新运行，一切归零
没有 memory 的 Agent ≈ 无状态的 REST API（每次请求独立，不记得上次调用）。
加上 memory 的 Agent ≈ 有状态 Session（记住用户偏好、上下文、历史操作）。

2. 三层记忆架构

层级存储内容生命周期类比
短期记忆当前对话的 messages 数组单次会话HttpServletRequest 属性
会话记忆对话摘要、关键实体单次会话（压缩后保留）HttpSession
长期记忆用户偏好、知识积累跨会话（持久化）MySQL 用户表

3. 短期记忆：messages 数组

这就是 L0004 一直在用的——messages 数组本身就是短期记忆。LLM 每轮都能看到完整历史。

问题：上下文窗口有限。GPT-4o 是 128K tokens，Claude Sonnet 是 200K。看似很大，但一个复杂 Agent 跑 20 轮可能就用完了。

短期记忆的核心挑战：上下文窗口溢出。
解决方案：
(1) 直接截断：只保留最近 N 轮（简单粗暴，丢失早期信息）
(2) 摘要压缩：把早期对话压缩成一段摘要（保留要点，丢失细节）
(3) 混合策略：最近 5 轮保留原文 + 更早的压缩成摘要
Claude Code 的 compaction 机制就是这个原理。 
def manage_messages(messages: list, max_recent: int = 5) -> list:
    """管理 messages：保留最近 N 轮 + 摘要"""
    if len(messages) <= max_recent * 2 + 1:
        return messages

    system_msg = messages[0]
    early = messages[1:-max_recent * 2]
    recent = messages[-max_recent * 2:]

    # 早期对话压缩成摘要
    summary = summarize_conversation(early)

    return [
        system_msg,
        {"role": "user", "content": f"[之前对话摘要]\n{summary}"},
        {"role": "assistant", "content": "好的，我记住了之前的对话内容。"},
        *recent,
    ]

def summarize_conversation(messages: list) -> str:
    """用 LLM 把历史对话压缩成摘要"""
    text = "\n".join(f"{m['role']}: {m['content'][:200]}" for m in messages)
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": f"请用 3-5 句话总结以下对话的关键信息：\n\n{text}"}],
        temperature=0,
    )
    return response.choices[0].message.content

4. 会话记忆：实体跟踪

会话记忆不只是摘要，还需要跟踪对话中的关键实体。

class SessionMemory:
    def __init__(self):
        self.entities = {}       # {"用户名": "张三", "项目": "Agent"}
        self.decisions = []      # 决策记录
        self.facts = []          # 已确认事实

    def get_context_string(self) -> str:
        parts = []
        if self.entities:
            parts.append("已知实体: " + ", ".join(f"{k}={v}" for k, v in self.entities.items()))
        if self.decisions:
            parts.append("已做决策: " + "; ".join(self.decisions[-3:]))
        if self.facts:
            parts.append("已确认事实: " + "; ".join(self.facts[-5:]))
        return "\n".join(parts) if parts else ""
会话记忆 ≈ HttpSession + @ModelAttribute。
短期记忆 ≈ request scope，会话记忆 ≈ session scope。
实体跟踪 ≈ 自动从表单绑定到 model——自动提取对话中的关键信息。

5. 长期记忆：跨会话持久化

import json
from pathlib import Path

class LongTermMemory:
    def __init__(self, user_id: str, storage_dir: str = "./memory"):
        self.path = Path(storage_dir) / f"{user_id}.json"
        self.path.parent.mkdir(exist_ok=True)
        self.data = self._load()

    def _load(self) -> dict:
        if self.path.exists():
            with open(self.path, encoding="utf-8") as f:
                return json.load(f)
        return {"preferences": {}, "knowledge": [], "summaries": []}

    def _save(self):
        with open(self.path, "w", encoding="utf-8") as f:
            json.dump(self.data, f, ensure_ascii=False, indent=2)

    def set_preference(self, key: str, value: str):
        self.data["preferences"][key] = value
        self._save()

    def add_knowledge(self, fact: str):
        self.data["knowledge"].append(fact)
        if len(self.data["knowledge"]) > 100:
            self.data["knowledge"] = self.data["knowledge"][-100:]
        self._save()

    def get_system_context(self) -> str:
        parts = []
        if self.data["preferences"]:
            prefs = ", ".join(f"{k}: {v}" for k, v in self.data["preferences"].items())
            parts.append(f"用户偏好: {prefs}")
        if self.data["knowledge"]:
            parts.append(f"已知知识: {'; '.join(self.data['knowledge'][-5:])}")
        return "\n".join(parts) if parts else ""

# 使用
ltm = LongTermMemory("user_zhangsan")
ltm.set_preference("language", "中文")
ltm.set_preference("tech_stack", "Java/Spring")
ltm.add_knowledge("用户在做一个 AI Agent 项目")
print(ltm.get_system_context())
# → 用户偏好: language: 中文, tech_stack: Java/Spring
#   已知知识: 用户在做一个 AI Agent 项目
长期记忆 ≈ 数据库用户表 + 偏好设置。
preferences ≈ user_settings 表，knowledge ≈ user_activity_log 表。
JSON 文件 ≈ 开发阶段 SQLite，生产换 MySQL/pgvector。

6. 三层记忆协作

def build_system_prompt(user_id: str, session_memory: SessionMemory) -> str:
    parts = [BASE_SYSTEM_PROMPT]

    # 长期记忆
    ltm = LongTermMemory(user_id)
    if ltm_ctx := ltm.get_system_context():
        parts.append(f"\n[长期记忆]\n{ltm_ctx}")

    # 会话记忆
    if session_ctx := session_memory.get_context_string():
        parts.append(f"\n[会话记忆]\n{session_ctx}")

    return "\n".join(parts)
"Agent 的记忆系统怎么设计？短期记忆和长期记忆有什么区别？"

回答要点：
(1) 三层架构：短期（messages）、会话（实体+摘要）、长期（持久化）
(2) 类比 Web：request scope → session scope → DB
(3) 短期记忆挑战是窗口溢出——用摘要压缩解决
(4) 长期记忆存用户偏好、知识积累、历史摘要
(5) 生产级可用向量库做语义检索（mem0 项目）

7. 自测题

题目 1：短期记忆溢出时，最佳策略是什么？

正确。混合策略最优：摘要保留早期要点，原文保留最近细节。Claude Code 的 compaction 就是这个原理。

题目 2：长期记忆应该存储在什么介质？

正确。长期记忆的核心是跨会话持久化。内存变量重启丢失，prompt 写死无法动态更新。文件/数据库是正确选择。

题目 3：三层记忆架构类比 Web 开发？

正确。短期随请求产生消失（request scope），会话跨请求保持（session scope），长期持久化到数据库。
