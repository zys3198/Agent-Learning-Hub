2. 问题 1：幻觉引用（Hallucinated Citations）

LLM 可能引用检索结果中根本不存在的内容。

检索结果: [来源1] "公司年假为 10 天。" [来源2] "入职满 1 年增加 2 天。"
LLM 回答: "公司年假为 15 天。[来源1][来源3]"  ← 15 天编的，[来源3] 不存在

解决方案：后置校验

import re

def verify_citations(answer: str, retrieved_chunks: list[dict]) -> dict:
    """校验回答中的引用是否真实存在"""
    cited = set(re.findall(r'\[来源 (\d+)\]', answer))
    available = set(str(i) for i in range(1, len(retrieved_chunks) + 1))

    hallucinated = cited - available
    if hallucinated:
        return {"valid": False, "error": f"回答引用了不存在的来源: {hallucinated}"}
    return {"valid": True}

result = verify_citations(answer, results)
if not result["valid"]:
    answer += f"\n\n⚠️ 注意：{result['error']}"

幻觉引用是信任问题。
两层防御：(1) Prompt 要求"只引用存在的来源" (2) 代码层校验引用编号。
类比：接口返回数据要做参数校验——不能信任上游（LLM）一定给正确数据。

def rag_with_fallback(question: str, index, chunks, threshold: float = 0.3) -> str:
    results = retrieve(question, index, chunks, top_k=3)

    if not results:
        return "抱歉，知识库中没有找到相关信息。"

4. 问题 3：重复调用工具

class ToolCallTracker:
    """跟踪工具调用历史，防止重复"""
    def __init__(self):
        self.history = []

    def is_duplicate(self, tool_name: str, args: dict) -> bool:
        import json
        args_key = json.dumps(args, sort_keys=True)
        return (tool_name, args_key) in self.history

    def record(self, tool_name: str, args: dict):
        import json
        args_key = json.dumps(args, sort_keys=True)
        self.history.append((tool_name, args_key))
重复调用 ≈ 前端重复提交。ToolCallTracker ≈ 幂等 token 检查——同样的请求只处理一次。
