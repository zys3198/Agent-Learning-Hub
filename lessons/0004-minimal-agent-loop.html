L0004 - 手写最小 Agent Loop

50 行代码写一个能跑的 Agent：calculator + search 两个工具

1. 回顾：Agent Loop 三要素

L0001 讲概念，L0002 讲 API 调用，L0003 讲 Tool Call。现在串起来：

Agent = LLM + Tools + Loop

while not done:
    response = llm.chat(messages, tools)       # ← L0002: API 调用
    if response.has_tool_calls:                # ← L0003: 解析 Tool Call
        results = execute_tools(tool_calls)    # ← L0003: 执行 + 回传
        messages.append(response, results)
    else:
        return response.content                # ← 终止：最终答案
Agent Loop ≈ 事件循环（Event Loop）。
每轮：从 LLM 拿一个"事件"（tool_call 或纯文本）。
tool_call → 执行 → 结果变新事件 → 继续循环。
纯文本 → 无新事件 → 循环结束。

2. 定义工具函数

import json

def calculator(expression: str) -> str:
    """计算数学表达式"""
    try:
        result = eval(expression)  # 仅用于演示，生产环境不要用 eval
        return str(result)
    except Exception as e:
        return f"计算错误: {e}"

def search(query: str) -> str:
    """模拟搜索（返回假数据）"""
    fake_db = {
        "Python": "Python 由 Guido van Rossum 于 1991 年发布。",
        "Java": "Java 由 Sun Microsystems 于 1995 年发布。",
        "Agent": "Agent 是能自主感知环境、做决策、执行动作的 LLM 应用。",
    }
    for key, value in fake_db.items():
        if key.lower() in query.lower():
            return value
    return f"未找到关于 '{query}' 的结果。"

4. Agent Loop 核心：不到 30 行

from anthropic import Anthropic

client = Anthropic()
MAX_STEPS = 5

def agent(user_input: str) -> str:
    messages = [{"role": "user", "content": user_input}]

    for step in range(MAX_STEPS):
        print(f"\n--- Step {step + 1} ---")

        # 1. 调 LLM
        response = client.messages.create(
            model="claude-sonnet-4-5-20250514",
            max_tokens=1024,
            system="你是一个助手。可以计算数学表达式和搜索知识库。用中文回答。",
            messages=messages,
            tools=tools,
        )

        # 2. 检查是否有 tool call
        if response.stop_reason == "tool_use":
            # 追加 assistant 的 tool_use 响应
            messages.append({"role": "assistant", "content": response.content})

            # 执行每个 tool call
            tool_results = []
            for block in response.content:
                if block.type == "tool_use":
                    print(f"  调用: {block.name}({block.input})")

                    # 执行工具
                    if block.name == "calculator":
                        result = calculator(block.input["expression"])
                    elif block.name == "search":
                        result = search(block.input["query"])
                    else:
                        result = f"未知工具: {block.name}"

                    print(f"  结果: {result}")
                    tool_results.append({
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": result,
                    })

            # 追加工具结果
            messages.append({"role": "user", "content": tool_results})

        else:
            # 无 tool call → 最终答案
            return response.content[0].text

    return "达到最大步数限制，Agent 终止。"

# 运行！
print(agent("Java 是什么时候发布的？它的发布年份乘以 2 是多少？"))

5. 运行效果

--- Step 1 ---
  调用: search({'query': 'Java 发布时间'})
  结果: Java 由 Sun Microsystems 于 1995 年发布。

--- Step 2 ---
  调用: calculator({'expression': '1995 * 2'})
  结果: 3990

--- Step 3 ---
Java 是在 1995 年由 Sun Microsystems 发布的。它的发布年份乘以 2 等于 3990。
看 Step 1 → Step 2：LLM 先搜到"1995"，然后用这个结果调 calculator。
这就是 Agent 和 Workflow 的区别——Step 2 的参数来自 Step 1 的结果，是 LLM 动态决定的，不是写死在代码里的。

6. messages 数组的变化过程

理解 Agent Loop 的关键：跟踪 messages 每一步的变化。

Step 0: messages = [
  {role: "user", content: "Java 是什么时候发布的？它的发布年份乘以 2 是多少？"}
]

Step 1 后: messages = [
  {role: "user", content: "..."},
  {role: "assistant", content: [tool_use: search("Java 发布时间")]},  ← LLM 决策
  {role: "user", content: [tool_result: "1995 年发布..."]},          ← 工具结果
]

Step 2 后: messages = [
  ... 上面的 3 条 ...,
  {role: "assistant", content: [tool_use: calculator("1995 * 2")]},
  {role: "user", content: [tool_result: "3990"]},
]

Step 3: stop_reason == "end_turn" → 返回最终文本
messages ≈ Servlet request 链。
每轮追加两条：LLM 决策（assistant）+ 工具结果（user/tool）。
下一轮 LLM 看到完整历史，所以能基于上一步结果做决策。

7. 自测题

题目 1：Agent Loop 的终止条件是什么？

正确。LLM 不再调工具时说明已能直接回答，循环结束。达到 MAX_STEPS 也会强制终止。

题目 2：为什么 Step 2 的 calculator 参数是 1995？

正确。LLM 看到 Step 1 结果"1995 年发布"，自主决定用 1995 调 calculator。参数来自中间结果，不是预定义的——这是 Agent 的核心特征。

题目 3：如果 MAX_STEPS 设为 1，上面的例子会怎样？

正确。MAX_STEPS=1 意味着循环只跑一次。LLM 还没来得及调 calculator 和生成最终答案。MAX_STEPS 要根据任务复杂度合理设置。
